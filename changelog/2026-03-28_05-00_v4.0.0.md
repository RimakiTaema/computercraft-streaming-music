# 4.0.0_vibe - Major UI Overhaul + Multi-Platform + Wireless Speakers + Web Dashboard

## 4-Tab Layout
- Player tab: clean now-playing view with full-height adaptive visualizer
- Queue tab: dedicated full-screen queue management with numbering
- Search tab: multi-platform search with platform icons
- Settings tab: playback controls, speaker list, changelog

## Smooth Visualizer
- Decoupled from audio chunk timing (was ~3fps, now 20fps)
- Interpolated rise/decay animation for buttery smooth bars
- Adaptive height: fills available space (3 rows on small, 20+ on big screens)
- Dedicated partial redraw to avoid full screen repaint at 20fps

## Adaptive Layout
- Three layout tiers: Compact (<40w), Normal (40-79w), Wide (80w+)
- Wide mode: single-row queue/search items, more info visible
- Compact mode: two-row format preserved for small screens
- Auto-scales to any monitor size

## Multi-Platform Search (API)
- SoundCloud: paste any SoundCloud URL, plays via yt-dlp
- Spotify: paste Spotify track/album URL, auto-matches to YouTube audio
- Direct URL: paste any audio URL supported by yt-dlp
- YouTube: unchanged, still primary platform
- Platform badges shown on player and search results

## Wireless Modem Speakers
- New receiver.lua script for remote speaker computers
- Main player auto-discovers network speakers via rednet ping
- Audio streamed over rednet protocol in real-time
- Works alongside local speakers (hybrid mode)
- Settings tab shows local + network speaker list with Scan button
- Can run with no local speakers if modem is attached

## Monitor Support
- Auto-detects attached monitor on startup
- Auto-scales text size to fit (tries 0.5 to 5.0)
- Uses monitor as primary display, terminal as fallback
- Monitor touch events work for all interactions
- Search input redirects to computer terminal

## Server Improvements
- yt-dlp fallback for search when RapidAPI quota runs out
- New ?url= parameter for non-YouTube audio downloads
- Platform detection: SoundCloud, Spotify, YouTube, direct URL
- Spotify oEmbed metadata lookup with YouTube audio matching
- All search results include platform field

## Bug Fixes
- Fixed multi-speaker overlap (unified retry loop)
- Fixed speaker delay (removed unnecessary wait-after-success)
- Fixed monitor touch not registering clicks
- Fixed play button state after song selection
- Fixed state reset on "Play now"
