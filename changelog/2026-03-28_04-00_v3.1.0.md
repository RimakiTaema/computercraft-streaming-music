# 3.1.0_vibe - Monitor Support + Bug Fixes

## Monitor Support
- Auto-detects attached monitor on startup
- Uses monitor as the primary display (not a mirror)
- Monitor uses 0.5 text scale for maximum detail
- Falls back to terminal if no monitor or monitor too small (< 29x12)
- Search input redirects to computer terminal with "Type here and press Enter" prompt
- No configuration needed, just attach a monitor

## Server Improvements
- yt-dlp fallback for search when RapidAPI quota runs out or fails
- Supports video lookup, playlist, and text search via yt-dlp
- Cookies support for yt-dlp authentication

## Bug Fixes
- Fixed play button showing wrong state after selecting a song from search
- Fixed UI not updating after adding songs to queue from search
- Fixed state not fully resetting when playing a new song via "Play now"
- Added explicit redraw after all search result menu actions
- Reset queue scroll position when replacing queue via playlist "Play now"
